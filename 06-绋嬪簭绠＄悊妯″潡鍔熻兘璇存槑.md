# 程序管理模块功能说明

## 一、模块概述

> **模块定位**：本模块为 SOP 操作规程管理模块（规程管理），用于管理设备操作规程、维护步骤、检查清单等标准化作业程序。支持版本管理、审批流程、步骤确认执行。
>
> **注意**：设备加工程序管理（NC/G代码/PLC程序下发）为未来扩展方向，不在当前版本范围内。

程序管理模块是DPMS系统中用于管理设备操作规程、维护步骤、检查清单等标准化作业程序的模块。该模块确保操作规程的版本可控、变更可追溯、执行可确认，保障作业过程的规范性和安全性。

## 三、页面与组件功能说明

### 3.1 程序库管理页面（ProgramLibraryPage）

#### 3.1.1 程序分类树组件（ProgramCategoryTree）
- **功能描述**：以树形结构展示程序的分类体系
- **核心功能**：
  - 树形数据懒加载，点击节点展开子分类
  - 支持拖拽排序，调整分类层级关系
  - 右键菜单：新增、编辑、删除分类

#### 3.1.2 程序列表组件（ProgramListPanel）
- **功能描述**：展示选定分类下的程序列表
- **核心功能**：
  - 表格展示：程序编号、名称、适用设备、当前版本、状态
  - 状态标识：草稿、待审批、已发布、已停用

#### 3.1.3 程序表单组件（ProgramFormDialog）
- **功能描述**：程序的创建和编辑表单
- **核心功能**：
  - 基础信息：程序编号、名称、分类、适用设备类型
  - 程序文件上传：支持NC、G代码、PLC程序等格式
  - 工艺说明：富文本编辑器录入加工工艺说明

### 3.2 版本控制页面（ProgramVersionPage）

#### 3.2.1 版本列表组件（VersionListPanel）
- **功能描述**：展示程序的所有历史版本
- **核心功能**：
  - 版本列表：版本号、版本说明、创建人、创建时间
  - 版本操作：查看、对比、回退

#### 3.2.2 版本对比组件（VersionComparePanel）
- **功能描述**：对比两个版本的程序内容差异
- **核心功能**：
  - 文件对比：文本模式对比程序代码差异
  - 差异导出：导出差异报告

#### 3.2.3 版本回退组件（VersionRollbackDialog）
- **功能描述**：将程序回退到指定历史版本
- **核心功能**：
  - 版本选择：选择要回退的目标版本
  - 回退原因：必填，记录回退原因

### 3.3 程序下发页面（ProgramDistributePage）

> ⚠️ **本节描述的功能为规划中/未实现，当前版本不包含程序下发功能。** 以下内容为未来规划，ProgramDistributePage、DistributeWizard、DistributeProgressPanel 及相关 API（POST /api/procedure/distribute/、GET /api/procedure/distribute/{id}/progress/）均未实现。

#### 3.3.1 下发向导组件（DistributeWizard）
- **功能描述**：引导用户完成程序下发到设备的完整流程
- **核心功能**：
  - 步骤一：选择程序版本
  - 步骤二：选择目标设备
  - 步骤三：配置下发参数
  - 步骤四：执行下发

#### 3.3.2 下发进度组件（DistributeProgressPanel）
- **功能描述**：实时展示程序下发的执行进度
- **核心功能**：
  - 进度总览：显示总体下发进度百分比
  - 设备列表：展示每台设备的下发状态
  - 重试机制：下发失败支持单独重试

### 3.4 文档解析与导入（ProcedureParseView）

- **功能描述**：支持从Word文档（.docx）中解析章节和步骤结构，并导入创建程序
- **核心功能**：
  - 解析方法列表：提供标准解析（standard）和智能解析（smart）两种模式
  - 标准解析：要求文档使用标准Word标题样式，校验通过后100%准确提取
  - 智能解析：综合检测样式、编号、字体等多维度信号，适用于非标准格式文档
  - 文档解析：上传.docx文件，自动提取章节层级和步骤内容
  - 程序导入：将解析结果导入创建程序，自动生成编号
- **后端实现**：`parser.py`（文档解析引擎）、`ProcedureParseView`（解析API）、`ProcedureImportView`（导入API）
- **解析策略**：auto/style_only/numbering_only/title_guided/content_blocks 五种策略

### 3.5 PDF生成服务（PDFService）

- **功能描述**：根据程序内容生成符合PPA AP-907-005 (Rev.2)规范的PDF文档
- **核心功能**：
  - PDF预览：生成PDF并返回base64编码，支持在线预览
  - PDF下载：生成PDF文件并下载，文件名格式：{code}_Rev{version}.pdf
  - 封面页：包含程序编号、名称、版本等基本信息
  - 目录页：自动生成章节目录
  - 修订页：展示版本修订历史
  - 内容页：章节和步骤的完整排版，支持警告块、签名栏等特殊元素
- **后端实现**：`pdf_generator.py`（PDF生成入口）、`pdf_cover_service.py`（封面/目录/修订页）、`pdf_chapter_service.py`（章节内容页）、`pdf_step_service.py`（步骤渲染）、`pdf_data_service.py`（数据组装）
- **页面规格**：A4纵向，上下边距1.27cm，左右边距2.03cm，宋体12pt

### 3.6 标记模式（MarkMode）

- **功能描述**：在程序编辑过程中，支持批量标记章节/步骤的内容类型
- **核心功能**：
  - 标记模式切换：开启/关闭标记模式
  - 批量选择：支持选择同一层级的章节进行批量操作
  - 标记为步骤（markAsStep）：将选中的内容块标记为步骤类型
  - 标记为文本（markAsContent）：将选中的内容块标记为文本类型
  - 单项标记循环：对单个步骤循环切换标记状态（未标记→步骤→文本）
  - 选择限制：仅支持同一层级选择，最多100项
- **前端实现**：`useMarkMode.ts`（标记模式composable）、`MarkModeToolbar.vue`（标记工具栏）
- **标记状态**：unmarked / step / content

### 3.7 章节步骤编辑器（ProcedureEditor）

- **功能描述**：提供程序章节和步骤的可视化编辑能力，支持树形结构管理
- **核心功能**：
  - 章节CRUD：新增、编辑、删除章节，支持多级嵌套（最多3级）
  - 步骤CRUD：新增、编辑、删除步骤，步骤归属于章节
  - 排序操作：章节/步骤上移、下移调整顺序
  - 类型转换：章节转步骤（convert-to-step）、步骤转章节（convert-to-chapter）
  - 内容拆分：内容块拆分为步骤（content-to-steps）、批量拆分
  - 根章节转步骤：将根级章节转换为独立步骤
  - 撤销/重做：支持编辑操作的撤销和重做
  - 编号自动更新：章节/步骤增删后自动重新编号
  - 会话存储：编辑状态自动保存到sessionStorage，防止意外丢失
- **前端实现**：`ProcedureEditor/index.vue`（编辑器主组件）、`ProcedureChapterTree.vue`（章节树组件）、`useProcedureEditor.ts`（编辑器composable）、`useProcedureEditorActions.ts`（编辑器操作composable）

### 3.8 程序审批页面（ProgramApprovalPage）

#### 3.8.1 审批列表组件（ApprovalListPanel）
- **功能描述**：展示待审批和已审批的程序变更申请
- **核心功能**：
  - 待办列表、已办列表、申请列表
  - 审批状态：待审批、审批中、已通过、已驳回

#### 3.8.2 审批详情组件（ApprovalDetailPanel）
- **功能描述**：展示程序变更申请的详细信息
- **核心功能**：
  - 申请信息、变更内容、审批流程
  - 审批操作：同意、驳回、转交、加签

## 五、API接口清单

### 5.1 程序管理接口
- `GET /api/procedure/programs/` - 程序列表查询
- `POST /api/procedure/programs/` - 创建程序
- `POST /api/procedure/programs/{id}/publish/` - 发布程序

### 5.2 版本管理接口
- `GET /api/procedure/programs/{id}/versions/` - 版本列表
- `POST /api/procedure/versions/{id}/rollback/` - 版本回退

### 5.3 文档解析与导入接口
- `GET /api/procedure/parse/methods/` - 获取可用解析方法列表
- `POST /api/procedure/parse/` - 解析上传的Word文档
- `POST /api/procedure/import/` - 导入解析结果创建程序

### 5.4 PDF生成接口
- `POST /api/procedure/procedures/{id}/pdf-preview/` - 生成PDF预览（返回base64）
- `GET /api/procedure/procedures/{id}/download-pdf/` - 下载PDF文件

### 5.5 章节管理接口
- `POST /api/procedure/chapters/` - 创建章节
- `PUT /api/procedure/chapters/{id}/` - 更新章节
- `DELETE /api/procedure/chapters/{id}/` - 删除章节
- `POST /api/procedure/chapters/{id}/move-up/` - 上移章节排序
- `POST /api/procedure/chapters/{id}/move-down/` - 下移章节排序
- `POST /api/procedure/chapters/{id}/convert-to-step/` - 将章节转换为步骤
- `POST /api/procedure/chapters/{id}/convert-to-content/` - 将章节转换为内容类型
- `POST /api/procedure/chapters/{id}/content-to-steps/` - 将内容拆分为步骤
- `POST /api/procedure/chapters/batch-content-to-steps/` - 批量将内容拆分为步骤

### 5.6 步骤管理接口
- `POST /api/procedure/steps/` - 创建步骤
- `PUT /api/procedure/steps/{id}/` - 更新步骤
- `DELETE /api/procedure/steps/{id}/` - 删除步骤
- `POST /api/procedure/steps/{id}/move-up/` - 上移步骤排序
- `POST /api/procedure/steps/{id}/move-down/` - 下移步骤排序
- `POST /api/procedure/steps/{id}/mark-status/` - 设置步骤标记状态

## 七、数据模型说明

### 7.1 Folder（文件夹）

| 字段名 | 类型 | 约束 | 默认值 | 业务含义 |
|--------|------|------|--------|----------|
| id | UUID | PK | auto | 主键 |
| name | CharField(100) | - | - | 文件夹名称 |
| prefix | CharField(20) | blank | '' | 编号前缀，用于程序编码生成 |
| parent | FK(Folder) | PROTECT, null, blank | None | 父文件夹，树形结构 |
| system | BooleanField | - | False | 是否系统文件夹（不可删除） |
| full_path | TextField | blank | '' | 完整路径缓存（如"父/子/孙"） |
| is_active | BooleanField | - | True | 是否启用（逻辑删除标记） |
| created_at | DateTimeField | - | auto_now_add | 创建时间 |
| updated_at | DateTimeField | - | auto_now | 更新时间 |

- 数据库表名：`tb_folder`
- 最大嵌套层级：`MAX_DEPTH = 5`
- 唯一约束：同一父文件夹下名称唯一（`uk_tb_folder_parent_name`）、full_path唯一
- 属性：`procedure_count`（文件夹下程序数）、`has_procedures`
- 方法：`can_delete()`（检查是否可删除）、`get_full_path()`

## 九、状态流转说明

### 9.1 Procedure状态流转

```
DRAFT ──→ PENDING ──→ APPROVED ──→ ARCHIVED
  │           │           │
  │           └──→ REJECTED   （驳回后可修改重新提交）
  │
  └──→ ARCHIVED   （草稿直接归档）

APPROVED ──→ ARCHIVED   （审批通过后可归档，支持自动归档）
```

- 初始状态：`DRAFT`
- 审批通过后进入`APPROVED`，此时`is_readonly=True`
- `ARCHIVED`为终态，不可修改

### 9.2 Procedure审批状态流转

```
DRAFT ──→ PENDING    （提交审批）
  │
PENDING ──→ APPROVED  （审批通过）
  │
  ├──→ REJECTED  （审批驳回）
  │
  └──→ DRAFT     （驳回后退回修改）
```

- 审批由`ProcedureSettings.enable_approval_workflow`控制
- 未启用审批时，发布直接进入APPROVED

### 9.3 ProcedureStep标记状态流转

```
pending ──→ completed   （步骤执行完成）
  │
  ├──→ skipped         （步骤被跳过）
  │
  └──→ pending         （重置为待处理）
```

## 十、业务规则补全

### 10.1 文件夹树形结构
- 最多5层嵌套（`MAX_DEPTH = 5`）
- 同一父文件夹下名称唯一
- 系统文件夹（system=True）不可删除
- 包含程序或子文件夹的文件夹不可删除
- 支持循环引用检测
- full_path自动缓存，修改时批量更新子文件夹路径

### 10.2 版本号管理
- 简单整数递增（1, 2, 3...），由`increment_version()`方法控制
- 每次变更记录日志到`version_change_log`（含change_type: update/publish/reject/restore）
- 最大版本号受`ProcedureSettings.max_version_number`限制
- 升级版本（upgrade-version）创建新Procedure记录
- 支持版本冲突检测（VERSION_CONFLICT返回409）

### 10.3 章节+步骤两级结构
- ProcedureChapter支持嵌套（parent字段），实现章节-子章节结构
- ProcedureStep通过chapter字段关联到章节
- 步骤也可直接属于程序（chapter=null）
- 支持：
  - 章节转步骤（convert-to-step）
  - 步骤转章节（convert-to-chapter）
  - 内容块拆分为步骤（content-to-steps）
  - 章节上移/下移排序

### 10.4 审批控制
- 由`ProcedureSettings.enable_approval_workflow`控制是否启用审批
- 启用时：DRAFT → 提交审批 → 工作流处理 → APPROVED/REJECTED
- 未启用时：可直接发布，跳过审批流程
- 审批模板来源：Procedure.approval_template 或 ProcedureSettings.default_approval_template
- 提交审批前必须关联审批模板
- 只有DRAFT状态的程序才能提交审批

### 10.5 步骤执行
- 步骤类型通过`input_schema`的JSON Schema定义
- 主要类型：
  - **pass_fail**：通过/不通过确认
  - **measurement**：测量值录入（可关联仪表，自动同步上下限）
  - **text**：文本记录
- `require_confirmation=True`时需要操作人员明确确认
- 步骤标记状态：pending → completed/skipped

### 10.6 程序复制
- `upgrade-version`：升级版本，创建新Procedure记录并复制内容
- 支持跨文件夹恢复（restore时指定target_folder_id）

### 10.7 自定义字段
- ProcedureField定义全局字段模板
- 支持类型：text/number/date/select/multi_select/checkbox/textarea
- 字段配置：required/default_value/options/validation_rules
- 字段状态管理：active/archived
- 支持排序（reorder）

### 10.8 附件管理
- 附件仅在规程（Procedure）级别管理，不绑定到章节或步骤
- 通过独立的附件模块关联（外键指向Procedure）

### 10.9 阅读确认
- 由`ProcedureSettings.require_read_confirmation`控制
- 启用时：用户需主动标记已读（mark-read）
- 可查询待阅读（pending-read）和已阅读（completed-read）列表

### 10.10 文档解析与导入
- 支持标准解析（standard）和智能解析（smart）两种模式
- 仅支持.docx格式
- 解析结果可导入创建程序，自动生成编号

### 10.11 PDF生成
- 支持在线预览（返回base64编码）
- 支持下载PDF文件（Content-Disposition: attachment）
- 文件名格式：{code}_Rev{version}.pdf

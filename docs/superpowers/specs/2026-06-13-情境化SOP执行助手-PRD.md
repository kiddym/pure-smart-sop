## 1. 背景与问题

现场设备维护按 SOP 执行，长期依赖人的记忆与自觉：
- **人因失误风险高**——漏步、错对象（开错阀门）、读错记错，缺实时纠偏；
- **执行痕迹不可追溯**——纸质或事后补录，谁/何时/做了什么说不清；
- **经验流失**——老师傅的窍门与坑点留不下、传不开；
- **SOP 缺陷难反馈**——现场发现的问题进不了改进闭环；
- **SOP 是孤立清单**——脱离上下游程序、设备档案、历史经验、相关读数，操作者要的信息不在手边。

## 2. 产品定位

**情境化 SOP 执行助手**：把 SOP 放回它的上下文网，在移动端**引导执行 + AI 监护防错 + 有据可依的实时建议 + 经验反馈闭环**。四支柱：①引导执行+电子记录 ②AI 监护防错（推）③情境聚合 ④操作助手 Copilot（拉），加经验闭环。详见 spec §1、D17。

### 目标
- 把 SOP 执行从"靠人自觉"变成"被引导 + 被监护 + 有据可问 + 留痕可溯"。
- 降低人因失误；执行记录满足 ISO 质量体系（受控/可追溯/签名，D8）。
- 沉淀并复用现场经验；现场发现反哺 SOP 改进。

### 非目标
- 不重建 SOP 编写系统（仅扩展元数据，§1 非目标）；不接 IoT/SCADA（读数靠人工/OCR，D20）；AI 不做物理硬拦（D4）；不做 IM、不创建工单（D28）。

## 3. 角色（Persona）

| 角色 | 在哪用 | 关注 |
|---|---|---|
| **操作者/维护技术员** | 移动端（现场，常戴手套/眼镜） | 要快、要简、防错、随手记、离线可用 |
| **班组长/主管** | Web | CR triage/处置/结案、异常复核 |
| **SOP 负责人/工艺工程师** | Web | 编写、监护期望标注、处置修正建议、改版 |
| **质量/持续改进岗** | Web | 经验审核精炼、报表、合规追溯 |
| **CMMS（系统）** | 集成 | 工单与身份来源、接收回写（D13/D30） |

## 4. 关键用户旅程

**旅程 A · 标准监护执行（高风险任务）**
张工在 CMMS 工单点"开始执行"→ 无缝调起执行运行时（D30）→ 戴眼镜逐步执行；某步手伸向 V2 → **红屏 + 震动 + 眼镜语音"停！应操作 V1"**（D32/D16）→ 纠正 → 读数口述 + 画面 OCR 双核（D3）→ 完成收尾屏逐条确认异常 + 签名 → 执行记录单推 Web 并回写工单（D31）。

**旅程 B · 纯手机离线执行（低风险点检）**
无眼镜、弱网；纯手机模式离线逐步打钩（D11）；某步长按参数记一条内联经验（D25/D29）；恢复网络自动同步、生成报告。

**旅程 C · 经验/改进闭环**
执行中长按参数"提修正建议"→ AI 整理成结构化草稿 → 推 Web（D29）；班组长据此或经验审核入库、或走 CR(SOP缺陷)→SOP 修订请求改版（D24）；监护偏差可一键升级 CR，结案蒸馏经验 → 喂 Copilot（D23）。

## 5. 功能需求（高层，实现见 spec）

- **引导执行 + 电子记录**：拆步执行、打钩、跳过/NA、交接、签名+PIN/工牌（D9/D15/D31）。
- **AI 监护防错**：眼镜采集+端侧实时核查（OCR/安全/物体/语义纠偏），三档告警呈现（D2/D12/D16/D32）。
- **情境聚合**：工单触发预取「执行上下文包」（本SOP+执行/编写参考SOP+图纸附件+设备档案+经验+历史读数），离线可读，构成 Copilot 边界（D18/D22）。
- **操作助手 Copilot**：带护栏 RAG，语音为主+预设问题，答案带引用、超边界拒答、离线降级检索（D19/D26）。
- **经验闭环**：内联经验（两级可见）+ Condition Report（轻量流程、CR→SOP 修订）+ 统一知识层（D23/24/25）。
- **移动端**：绝对简洁（单步聚焦+上滑上下文条、多步大纲、长按操作面板），双模式入口，完成→报告推送（D27–D31）。
- **Web 端**：编写/监护期望标注/参考关系图纸维护、CR 处置、经验审核、报表、配置（§4.1）。
- **CMMS 集成**：执行运行时复用，契约优先，工单绑定+回写（D30）。

## 6. 非功能需求（NFR，合并）

| 维度 | 要求 | 来源 |
|---|---|---|
| 合规 | ISO 体系：记录受控+可追溯到人+留存可配；轻量签名+PIN/工牌 | D8/D9 |
| 性能 SLO | 端侧实时核查 <1s 且**弱网也出**；云 VLM 难判尽力实时、弱网当场转人工；Copilot 在线几秒、离线检索原文 | §9/D11/D26 |
| 离线 | 纯手机模式**全程离线可执行**，本地队列恢复自动同步 | D11/D18 |
| 隐私 | 手机端脱敏、默认只模糊人脸、按需录、媒体受控留存 | D14 |
| AI 安全 | 全为建议、人决定、标来源、超边界拒答；Copilot 只引用已审核经验 | D4/D19/D25 |
| 可用性 | 绝对简洁、语音优先、大字/手套/强光、防误触 | D26/D27 |

## 7. 成功指标（KPI，目标值待校准）

- **防错**：关键步骤偏差当场告警覆盖率↑；**漏报率上限**；**监护误报率上限**（防警报疲劳，关键体验指标）。
- **合规**：步骤完整执行率↑；记录可追溯率 = 100%；报告自动生成率↑。
- **经验闭环**：CR 闭环时长↓；已审核经验条目增长；Copilot 建议采纳率。
- **效率**：执行记录电子化率；事后补录耗时↓。
- **采纳**：一线活跃使用率；内联经验/修正建议提交量。

## 8. MVP（第一期）范围与验收

**范围**（对应 spec 分期第一期）：纯手机执行+电子记录全链路；端侧 OCR+安全合规两类实时核查+眼镜串流；云 VLM 少量难判+弱网转人工；三档告警；上下文包基础（本SOP+执行参考SOP+图纸+设备档案，离线可读）；内联经验录入；完成→报告推送+回写。

**验收标准（可测）**：
1. 工单发起 → 逐步执行 → 打钩签名 → 生成执行记录单推 Web 并回写工单，端到端跑通；
2. 端侧 OCR + 安全合规两类核查实时出结果，**断网时仍出**，按三档正确呈现；
3. 纯手机模式**全程离线**可执行，恢复网络自动同步无丢失；
4. 上下文包基础内容**离线可读**；内联经验可录入并按两级可见性处理；
5. 执行记录满足 ISO：受控、可追溯到人、签名核验、留存可配。

## 9. 假设 / 依赖 / 风险

- **依赖**：CMMS 工单/设备档案/身份(SSO)接口可用（D9/D13/D30）；智能眼镜选型及低延迟本地串流能力（§14.6/开放#4）；手机端 NPU 算力（D12）。
- **假设**：读数靠人工/OCR、不接 IoT（D20）；身份来自 CMMS/SSO（D9）。
- **风险**：
  - 端侧模型精度与**误报率**——需采集数据 + 评测基线（开放#7/#12）；
  - 眼镜采购/续航/佩戴体验（开放#4）；
  - Copilot 幻觉——靠 D22 边界 + 护栏 + 评测，未达标不放（开放#12）；
  - 现场噪声影响语音——预设问题兑底（D26）；
  - 长按手势可发现性——首次轻提示（D29）；
  - 知识库早期薄——bootstrap，自动沉淀+专家审核逐步养（D21/D23）。
